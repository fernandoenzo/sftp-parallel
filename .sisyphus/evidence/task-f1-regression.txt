## 1. BUGS FOUND AND FIXED IN v7
================================================================

### Bug 1: collect_results subshell issue (CRITICAL)
- CAUSE: `FAILED=$(collect_results PIDS "$WORK_DIR")` creates a subshell
  via command substitution `$(...)`. The `wait` command inside the subshell
  cannot wait for background processes from the parent shell, causing all
  SFTP sessions to appear as failed (exit -1 / signal 1).
- FIX: Changed `collect_results` to write to global `FAILED` variable directly
  instead of `echo`ing. Changed call site from `FAILED=$(collect_results PIDS ...)`
  to `collect_results "$WORK_DIR"`.
- Also removed `local -n _pids=$1` nameref since the global `PIDS` array
  is used directly.

### Bug 2: create_batch empty filename (MEDIUM)
- CAUSE: `mapfile -t _bucket_files <<< "$_bucket_contents"` produces an empty
  trailing element when the bucket string ends with a newline. This empty
  string was passed to `create_batch` which generated `put -f "/local/dir/"`
  causing SFTP to fail with "not a regular file" error.
- FIX: Added `[[ -n "$FILE" ]] || continue` in the create_batch loop.

### Bug 3: collect_results log output (LOW)
- CAUSE: Line 174 originally was `<"$_work_dir/out_${_i}.log" >&2` which
  redirects stdin from file and stdout to stderr but with no command, making
  it a no-op. V6 used `cat "$TMPDIR/out_${I}.log" >&2`.
- FIX: Changed to `cat "$_work_dir/out_${_i}.log" >&2`.
