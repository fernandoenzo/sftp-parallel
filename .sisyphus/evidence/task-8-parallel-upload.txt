Task 8: Implement parallel upload with Popen process management

== Implementation ==

Added `run_parallel_uploads(host, buckets, timeout)` to src/sftp_parallel/uploader.py:
- Takes host, list of file-list buckets (from distribute_files), and timeout
- Spawns one subprocess.Popen per non-empty bucket using sftp -N -o ConnectTimeout=X -o BatchMode=yes -b - host
- Passes batch commands via proc.communicate(input=batch_commands)
- Stores (process, batch_commands) pairs to ensure correct commands per process
- Filters out empty buckets before spawning processes
- Returns early with (True, 0) for empty/blank buckets
- Counts failures (nonzero returncode or communicate exception)
- Returns (all_success: bool, failed_count: int)

== Tests ==

Added 9 new tests in tests/test_uploader.py:

TestRunParallelUploadsAllSucceed:
- test_all_succeed: verifies all succeed with returncode=0
- test_communicate_receives_batch_commands: verifies each process gets correct batch commands
- test_sftp_command_includes_host_and_options: verifies cmd construction with timeout

TestRunParallelUploadsSomeFail:
- test_some_fail: one succeeds, one fails -> success=False, failed=1
- test_all_fail: both fail -> success=False, failed=2
- test_communicate_exception_counts_as_failure: OSError in communicate -> failure

TestRunParallelUploadsEmptyBuckets:
- test_empty_buckets_skipped: all empty -> success=True, failed=0, no Popen calls
- test_mixed_empty_and_nonempty: mixed -> only 2 Popen calls for non-empty
- test_completely_empty_list: empty list -> success=True, no Popen calls
